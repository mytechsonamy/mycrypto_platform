VALIDATION RULES
================================================================================

PHASE 1: Market Data
  • Status codes: 200 OK
  • Bids sorted: Descending (highest first)
  • Asks sorted: Ascending (lowest first)
  • Bid-ask spread: Exists and valid
  • Response times: <100ms
  • Data types: Decimals for prices, integers for timestamps
  • Pagination: Total, limit, offset included

PHASE 2: Order Placement
  • Status codes: 201 Created
  • Market orders: Status FILLED or PENDING
  • Limit orders: Status PENDING
  • Invalid input: Status 400 with error message
  • Insufficient funds: Status 400/422
  • Balance validation: Prevents overspending

PHASE 3: Order Management
  • Status codes: 200 OK, 204 No Content
  • Open orders: Only PENDING or PARTIALLY_FILLED
  • Cancellation: Immediate and confirmed
  • Fund release: Within 100ms of cancellation
  • History: Complete with proper filtering
  • WebSocket: Real-time updates <100ms

PHASE 4: History & Analytics
  • Trade records: Complete and accurate
  • P&L math: Correct calculations verified
  • Exports: CSV and JSON formats valid
  • Settlement: Status tracked properly
  • Pagination: Works correctly with limits

PHASE 5: Advanced Features
  • Alerts: Created successfully (201)
  • Indicators: Calculated per formula
  • Signals: Based on multiple indicators
  • Webhooks: Deliver within 5 seconds
  • Notifications: Real-time via WebSocket

PHASE 6: Performance
  • Latency: Within targets (p50, p95, p99)
  • Throughput: Handles 100+ concurrent
  • WebSocket: Stable without disconnects
  • Message ordering: Sequence maintained
  • Data integrity: No loss or duplication
